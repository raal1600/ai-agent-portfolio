From job posting to ready-to-send PDFs.

A readable transcript of the seven-step SoherDocs product-concept recording.

Synthetic product concept. This illustrates intended workflow behavior, not a recording of shipped software or runtime evidence. The person, company, job posting, profile evidence, documents, and ATS score are fabricated.

  1. Add the job posting
  2. Find profile evidence
  3. Tailor the CV
  4. Write the cover letter
  5. Generate and check ATS readiness
  6. Optionally save the CV
  7. Export both PDFs
Step 1 · 00:00–00:05

Add the job posting

Maya starts a new application by pasting a role or uploading the original job-post PDF.

Input
A synthetic AI Product Engineer posting from Northstar Labs.
Action
Select “Analyze job & find matches.”
Result
The role text is ready for requirement extraction and profile matching.
Step 2 · 00:05–00:10

Find the strongest profile evidence

AI extracts the job requirements, searches six saved CV variants, and shows where each matching fact came from.

Input
The job posting and Maya’s profile knowledge base.
Action
Compare required skills with evidence saved across CV variants.
Result
Nine profile-backed matches, including Python APIs, LLM evaluation, product collaboration, and English proficiency.
Step 3 · 00:10–00:15

Build a role-matched CV

The strongest saved evidence is paired with the role requirements and emphasized in a custom CV draft.

Input
Job requirements and matched profile evidence.
Action
Select relevant experience and skills for this role.
Result
A custom CV preview that uses profile-backed examples without inventing experience.
Step 4 · 00:15–00:20

Write a personal cover letter

Profile knowledge, job context, and personal details are combined into one tailored letter that the user can review.

Input
Saved skills and experience, the job posting, plus Maya’s motivation and preferred tone.
Action
Connect the most relevant evidence to the company’s stated needs.
Result
A personal cover-letter draft with its sources and personalization choices explained.
Step 5 · 00:20–00:25

Generate and check ATS readiness

Both documents are generated, then the CV receives an illustrative ATS-readiness score with checks and an actionable improvement.

Input
The tailored CV and cover letter.
Action
Check keywords, document structure, formatting, and role evidence.
Result
An illustrative 88/100 readiness score and one advisory improvement.
Step 6 · 00:25–00:30

Optionally save the custom CV

Maya chooses whether to keep this role-specific CV as a new knowledge-base variant. Skipping does not block export.

Input
The completed custom CV.
Action
Select “Not now” or “Save as a new variant.”
Result
In this walkthrough, a seventh CV variant is added while the six existing versions remain unchanged.
Step 7 · 00:30–00:35

Export both PDFs directly

The tailored CV and cover letter are presented as separate PDF downloads, ready for the user’s final review and application.

Input
The generated application package.
Action
Select “Download CV” and “Download letter.”
Result
Two proposed PDF exports. The portfolio concept does not generate or download actual files.
