# Guided demo — SoherDocs

The published WebM is a captioned architecture walkthrough generated from the sanitized public case study. It is not a recording of private accounts, profile data, job applications, or infrastructure.

## Suggested supervised live demonstration (8–10 minutes)

1. **Sign in with a synthetic account.**
2. **Create or edit a synthetic profile** and explain structured ownership.
3. **Add a synthetic job description** and show local analysis.
4. **Preview a tailored document** without persisting private raw input unexpectedly.
5. **Export HTML/PDF/text** and explain the replaceable renderer/storage boundaries.
6. **Show an AI-assisted suggestion** and the explicit review/apply step.
7. **Run focused tests** covering ownership, rendering, and one browser flow.
8. **Handover:** show the versioned plan, quality gates, and operational boundaries.

## Privacy checklist

- Use synthetic names, contact details, and job text.
- Use non-production storage and database configuration.
- Hide credentials, provider configuration, private repository details, and real application artifacts.
- Clear generated test artifacts after the session.
