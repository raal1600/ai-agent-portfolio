Agent engineering

Soher agent system

Soher is my custom governed delivery system built on the Pi framework. The parent Pi session coordinates normal single-target phases and every canonical workflow mutation. Only an explicitly authorized parallel group may delegate isolated code work to yolo-implementer workers or fresh read-only review to yolo-reviewer workers.

My contribution

Designed and implemented as an independent project

I designed the control model, implemented the TypeScript/Node.js extensions, integrated framework lifecycle events and Obsidian tooling, and built the test and recovery strategy.

  • Defined capability, authority, phase, remediation, and hard-stop contracts.
  • Implemented dispatch gates, durable session continuity, cross-process leases, and bounded parallel execution.
  • Built filesystem-first knowledge tooling, safety checks, regression suites, and operator-facing controls.
Why it matters

Making AI-assisted delivery dependable

Authority stays explicit

Agents execute only within a user-authorized batch, ticket, phase, workspace, and security boundary.

Concurrency stays controlled

Cross-process leases prevent duplicate ownership while bounded isolated agents can work in parallel.

Failure stays recoverable

Durable checkpoints, reconciliation, bounded remediation, and hard stops prevent silent continuation.

A controlled agent workflow

  • Commits, deployment, destruction, and next-batch progression remain outside automatic authority.
